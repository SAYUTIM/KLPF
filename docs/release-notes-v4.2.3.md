# KLPF v4.2.3 修正メモ

GitHubへ公開するときの説明文に転用するための作業メモです。

## 課題リストとホーム出席表示の競合を修正

- ホーム画面の再読み込み直後に課題を開くと、KU-LMSのシステムエラーページへ遷移する可能性がある問題を修正しました。
- 原因は、ホーム出席表示が講義ごとの出席状態を確認するPOSTと、課題ページへ遷移するPOSTが、KU-LMSのサーバー側セッションにある「現在の講義」情報を同時に変更していたことでした。
- 固定時間だけ課題をクリック不能にする暫定対策を廃止しました。
- 課題クリック時は、実行中の出席判定リクエストが完了するまで待ち、課題一覧のサーバー側コンテキストを明示的に復元してから課題へ遷移するようにしました。
- 課題一覧のバックグラウンド更新で使用するiframe、MutationObserver、タイマー、Promiseの後片付けを整理しました。
- ページ離脱やユーザー操作時に、不要になった課題更新処理が残らないようにしました。

## 講義カードの誤遷移を修正

- ホーム画面の表示直後に「ソフトウェア工学I」などの講義を開くと、先頭の「哲学B」へ誤って遷移する可能性がある問題を修正しました。
- 「履修中科目のみ表示」の自動フィルターが有効な状態で再現しましたが、直接原因はフィルターではなく、ホーム出席表示の最初の出席判定POSTと講義カードの遷移POSTの競合でした。
- ホーム出席表示のスクリプトを `document_start` から起動し、ページ表示直後から講義カードの操作を監視するようにしました。
- 講義カードが押された場合は出席判定の新規リクエストを止め、実行中のリクエストが完了した後、ユーザーが選択した講義IDを最後に送信して遷移するようにしました。
- クライアント側で通信を中断するだけではなく、サーバー側セッションの更新順序を保証する方式へ変更しました。

## 関連する安定性改善

- ホーム出席表示のイベントリスナーが重複登録されないようにしました。
- 出席判定、課題更新、課題遷移、講義遷移の状態を明示的に管理するようにしました。
- 任意の待ち時間に依存する処理を、実際の非同期処理の完了を待つ方式へ変更しました。
- 自動フィルターのチェックボックスを、入れ子のlabelを使わない正しいDOM構造へ変更しました。
- 保存されたフィルター設定の型を検証し、壊れた設定値で初期化全体が失敗しにくいようにしました。
- 「履修中科目のみ表示」の機能トグルをONにしたときだけ、「自動」チェックもONにするようにしました。その後ユーザーが「自動」をOFFにした場合は、その設定を維持します。
- 現在授業の赤枠判定に学期／クォーター条件を追加し、同じ曜日・時限にある過去・未来の科目が同時に赤くなる問題を修正しました。
- 課題通知がOFFでも送信処理を呼び出していた真偽判定を修正しました。
- GAS Webhook URLを送信直前にバックグラウンド側で検証するようにしました。

## ホームダッシュボード編集と課題カレンダー

- ホーム左カラムに編集ボタンを追加し、中央パネルからプロフィール、学習実績、利用状況、課題カレンダー、課題リストの表示・非表示と順序を変更できるようにしました。
- 編集中はホーム画面全体を暗いオーバーレイで覆い、編集パネル以外を操作できないことが分かるようにしました。
- FullCalendar 6.1.21を拡張機能内へ同梱し、課題提出日に授業科目名を表示するカレンダーを追加しました。
- 左カラムでは境界線を抑えたミニ月表示を追加し、元サイトの背景色やプロフィール領域の連続性を維持するデザインにしました。
- 課題ごとに非表示ボタンを追加し、再読み込み後も非表示状態を維持するようにしました。
- 非表示にした課題はホーム編集パネルから個別に復元できるようにしました。
- 非表示課題を復元候補から完全に削除し、再取得後も表示しない操作を追加しました。
- 課題カレンダーを日曜始まりにし、24:00期限をLMSに表示された日付の予定として扱うようにしました。
- ミニカレンダーは1日あたり最大3個の点と残件数、拡大表示は最大3件と残件ポップオーバー／日別詳細で、多数の課題も確認できるようにしました。
- ホーム編集画面を大きくし、月の日数にかかわらず6週分を固定表示して、月移動でパネルの高さが変わらないようにしました。
- ホーム編集ボタンをサイドメニュー下部へ移動し、課題の更新表示を課題リストの先頭へ統合しました。
- LMSのユーザーメニューに「サイトテーマ色変更」を追加し、科目名とお知らせリンクの色をカラーピッカーまたは16進数で変更・保存・既定色へ復元できるようにしました。
- スマートフォン用ユーザーメニューの誤った階層へ設定項目が挿入され、ドロップダウンが常時表示される問題を修正しました。
- テーマ色設定を文字色、科目カード枠線、科目情報背景、お知らせ背景の4項目へ分け、個別のライブプレビューと既定色復元に対応しました。
- ホーム編集画面を低いディスプレイにも追従させ、カレンダー下端が潰れないようにしました。一覧表示では「終日」の代わりにLMS上の提出時刻を表示します。
- ホーム編集やKLPF設定を開いている間は背景ページのスクロールを固定し、ポップアップ内だけをスクロールするようにしました。
- サイドメニュー下部のホーム編集ボタンを廃止し、ユーザーメニューの「KLPF 設定」直下へ「ホームの表示を編集」を追加しました。
- テーマ編集を右下のドラッグ可能な小型ツールへ刷新し、サイト上でクリックした要素の文字色・背景色・枠線色を直接変更できるようにしました。直近5色の再利用にも対応しました。
- テーマツールのドラッグを描画フレーム同期の`transform`方式へ変更し、移動時のカクつきを抑えました。
- 要素選択中は透明な検査レイヤーでサイト操作を遮断してリンク遷移を防ぎ、選択後の階層チップから内側・親要素を直接切り替えられるようにしました。
- 最近使った色は保存時の確定色だけを最大5色記録し、ツールの横幅いっぱいに表示するようにしました。
- テーマツールでは背景ページをスクロール可能にし、属性選択を文字・背景・枠線のボタン形式へ変更しました。
- 選択要素を既定色へ戻せない問題を修正し、テーマ設定内の説明文を削減しました。
- 個別の色指定を戻す操作に加え、保存色と個別ルールをまとめて初期化する全体復元を追加しました。
- 色履歴をカラーピッカーの変更確定時に即時保存し、再読み込み後も維持するようにしました。同じ色は重複登録せず、ホバー時に16進数を表示します。
- 個別復元と全体復元の配置を入れ替え、全体復元を「すべて規定色に戻す」へ変更しました。全体復元では色履歴を変更しません。
- コンテナの文字色を子要素にも適用し、枠線がない要素へ枠線色を指定した場合は1pxの実線を追加するようにしました。
- 親要素と子要素の文字色ルールをDOM階層順に適用し、子要素へ個別指定した色が親要素の変更で上書きされないようにしました。
- 教材・トピックからホームへ戻る `/lms/tpicTpil/doBack` でも、課題一覧、ホーム出席表示、履修科目フィルターが動作するようにしました。
- TOTPの期限・大学側の設定・KLPFへの取込方法をまとめた案内ページを追加し、自動ログイン設定から参照できるようにしました。
- 認証システムのTOTP用QR画面を検出し、利用者の確認後に秘密鍵をKLPFへ保存できるフローティング案内を追加しました。
- 課題リストの背景・枠線・影を取り除き、左カラム内で浮いて見えない表示へ調整しました。
